The LLMR ecosystem

Language-model tooling for research in R: one provider layer, and a methodology stack built on it — validated annotation, robustness audits, verifiable replication archives, agents, and calibrated silicon samples. Open-weight models brought the cost of rigor down to pennies; these packages make the rigorous path the easy one.

Why this exists

Language models now produce measurements that reach published tables, and practice has moved faster than method: a single prompt and a single run often stand in for an instrument, with no codebook, no reliability estimate, and no record a later reader could consult. Content analysis settled these questions decades ago — instruments, gold standards, reliability, documentation. These packages carry that discipline over to LLM-assisted work and, by leaning on inexpensive open-weight models, make it cheap enough to be routine.

The workflow, end to end

  • callLLMR
    any provider, one interface
  • simulateLLMRAgent · LLMRpanel · FocusGroup
    agents, panels, groups
  • measureLLMRcoder
    codebooks, sealed gold
  • auditLLMRvalid
    the measurement multiverse
  • archiveLLMRarchive
    reviewer-runnable provenance

Which package?

PackageUse it whenNot for
LLMRcoder Validated text measurement for quantitative inference Accessible qualitative coding or text segmentation
LLMRvalid Robustness of a downstream estimand across measurement choices Construct validity or pass/fail validation
LLMRarchive Reviewer-runnable replication archives from LLM audit logs Measurement or estimation
LLMRpanel Calibrated silicon survey samples for design-stage work Human-population estimates without calibration to a human benchmark

All four packages follow the same workflow contract: construct a first object, build or extend it, run it with a .runner seam, read diagnostics(), draft report(), and optionally archive with LLMRarchive.

The packages

LLMR

CRAN

Unified access to OpenAI, Anthropic, Gemini, Groq, DeepSeek, Together, Ollama and more: structured output with schema enforcement, tool calling, streaming, logprobs, embeddings, tidy data-frame verbs, parallel and half-price batch execution, audit logging, replication and agreement statistics, cost accounting.

Site GitHub

LLMRAgent

GitHub

Agents with personas, tools, pluggable memory, and hard budgets checked before every call; agents that delegate to other agents; multi-agent debates, interviews, and deliberations with private votes; factorial agent experiments; a strong-planner/cheap-worker orchestrator.

Site GitHub

LLMRcoder

GitHub

Codebook-first annotation: versioned, hashable instruments; gold sets whose sealed test split records every evaluation in a visible ledger; locked protocols; prompt-and-model tournaments with cost columns; replicated corpus coding; generated methods text. gold_correct() reports measurement-error-corrected category prevalences from the audited test split, with standard errors.

Site GitHub

LLMRvalid

GitHub

Specification-curve analysis where the specifications are measurement choices: the estimand as code, recomputed across prompts, model families, label order, and temperature; sign and magnitude stability; a fragility index for how little must change before the conclusion flips. No badges, deliberately.

Site GitHub

LLMRarchive

GitHub

Replication archives built from audit logs: content-addressed records, one sealed root to cite, completeness linting that maps every reported number to a logged call, IRB-grade redaction that keeps the hash tree while withholding the text, and a verifiability horizon that says which calls will still be checkable in ten years.

Site GitHub

LLMRpanel

GitHub

Calibrated silicon samples for the design stage: persona panels from margins you supply, Likert and choice items, vignette and conjoint designs with order randomization recorded as data, bias audits, and a banner that refuses to disappear until benchmark comparison earns it.

Site GitHub

FocusGroup

GitHub

Simulation and analysis of focus-group discussions with LLM agents: moderated sessions, turn dynamics, and transcript analysis for instrument piloting and theory development.

Principles

  • Validation as the default path. Codebooks, gold sets, and model tournaments cost cents on open-weight models; the tooling makes them fewer keystrokes than their absence.
  • Complete records. Every call can be logged, hashed, sealed, and replayed, so a methods section can cite an artifact rather than a model name.
  • Diagnostics in the objects. Test-split evaluations are ledgered, uncalibrated panels are labeled as such, and estimates carry a fragility index — the state of the evidence travels with the result.
  • Wide access. A small lab runs the same audits as a well-funded one, and local open weights keep restricted text on the machine where it lives.
