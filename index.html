The LLMR ecosystem

Language-model tooling for research in R: one provider layer, and a methodology stack built on it — validated annotation, robustness audits, verifiable replication archives, agents, and calibrated silicon samples. Open-weight models brought the cost of rigor down to pennies; these packages make the rigorous path the easy one.

Why this exists

LLMs are already producing measurements that end up in published tables — too often from one prompt, one model, one run, with no codebook, no gold set, no log, and no way for anyone to check. The ecosystem takes the opposite bet: validation discipline borrowed from content analysis, provenance an editor could demand by name, and economics that no longer excuse skipping either. Software cannot prevent dishonesty; it can make honesty cheap and dishonesty visible.

The packages

LLMR

CRAN · the foundation

Unified access to OpenAI, Anthropic, Gemini, Groq, DeepSeek, Together, Ollama and more: structured output with schema enforcement, tool calling, streaming, logprobs, embeddings, tidy data-frame verbs, parallel and half-price batch execution, audit logging, replication and agreement statistics, cost accounting.

Site GitHub

LLMRAgent

GitHub

Agents with personas, tools, pluggable memory, and hard budgets checked before every call; agents that delegate to other agents; multi-agent debates, interviews, and deliberations with private votes; factorial agent experiments; a strong-planner/cheap-worker orchestrator.

Site GitHub

LLMRcoder

GitHub · in development

Codebook-first annotation: versioned, hashable instruments; gold sets whose sealed test split records every evaluation in a visible ledger; locked protocols; prompt-and-model tournaments with cost columns; replicated corpus coding; generated methods text. Measurement-error correction via the ipd bridge arrives in 0.2.

Site GitHub

LLMRvalid

GitHub · in development

Specification-curve analysis where the specifications are measurement choices: the estimand as code, recomputed across prompts, model families, label order, and temperature; sign and magnitude stability; a fragility index for how little must change before the conclusion flips. No badges, deliberately.

Site GitHub

LLMRtrail

GitHub · in development

Replication archives built from audit logs: content-addressed records, one sealed root to cite, completeness linting that maps every reported number to a logged call, IRB-grade redaction that keeps the hash tree while withholding the text, and a verifiability horizon that says which calls will still be checkable in ten years.

Site GitHub

LLMRpanel

GitHub · in development

Calibrated silicon samples for the design stage: persona panels from margins you supply, Likert and choice items, vignette and conjoint designs with order randomization recorded as data, bias audits, and a banner that refuses to disappear until benchmark comparison earns it.

Site GitHub

FocusGroup

GitHub

Simulation and analysis of focus-group discussions with LLM agents: moderated sessions, turn dynamics, and transcript analysis for instrument piloting and theory development.

Principles

  • Validation is the default path. Codebooks, sealed gold sets, and tournaments cost cents on open-weight models; the tooling makes them fewer keystrokes than skipping them.
  • Provenance over assertion. Every call can be logged, hashed, sealed, and replayed; "we used GPT-4" stops being a methods section.
  • Honesty by visibility. Test-split peeks are ledgered, uncalibrated panels carry banners, fragile conclusions get an index. The objects tell on themselves, politely.
  • An equalizing force. The lab with fifty dollars runs the same audits as the lab with fifty thousand; local open weights keep IRB-restricted text on the machine it lives on.
